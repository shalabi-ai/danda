# Danda Accessor Reference

The Danda accessor extends every pandas `DataFrame` with the `.dg` namespace, providing a consistent API for cleaning, analyzing, optimizing, and imputing data.

```python
import pandas as pd
import danda

df = pd.read_csv("customers.csv")

df = (
    df
    .dg.clean()
    .dg.impute()
    .dg.optimize()
    .dg.analyze()
)
```

Each method returns a new `DataFrame`, allowing operations to be chained naturally while preserving DataFrame metadata.

---

# `clean()`

Clean a DataFrame by applying a sequence of data cleaning plugins.

```python
df = df.dg.clean()
```

## Description

The cleaning pipeline removes or normalizes common data quality issues.

The following plugins are executed in order:

| Plugin | Description |
|---------|-------------|
| EmptySpacesPlugin | Removes leading and trailing whitespace from string values. |
| NormalizeMissingValuesPlugin | Converts configured missing-value indicators (for example `"N/A"` or `"-"`) into `NaN`. |
| SparseColumnsPlugin | Removes columns exceeding the configured missing-value threshold. |
| SparseRowsPlugin | Removes rows exceeding the configured missing-value threshold. |
| EmptyRowsPlugin | Removes rows containing only missing values. |
| EmptyColumnsPlugin | Removes columns containing only missing values. |
| DropDuplicatesPlugin | Removes duplicate rows. |

## Returns

A cleaned `DataFrame`.

## Report

The cleaning report is stored in

```python
df.attrs["danda_clean_report"]
```

and can also be accessed through

```python
df.dg.report
```

---

# `optimize()`

Optimize DataFrame memory usage by converting columns to more efficient data types.

```python
df = df.dg.optimize()
```

## Description

The optimization pipeline automatically converts eligible columns to memory-efficient pandas types.

Plugins executed:

| Plugin | Description |
|---------|-------------|
| BooleanTypePlugin | Converts boolean-like columns to `bool`. |
| DateTimeTypePlugin | Converts datetime-like columns to `datetime64`. |
| NumericTypePlugin | Downcasts numeric columns where appropriate. |
| CategoryTypePlugin | Converts low-cardinality string columns to `category`. |

## Returns

An optimized `DataFrame`.

## Report

Stored in

```python
df.attrs["danda_optimize_report"]
```

---

# `analyze()`

Analyze a dataset without modifying it.

```python
df = df.dg.analyze()
```

## Description

Analysis plugins generate data quality reports while leaving the original data unchanged.

Plugins executed:

| Plugin | Description |
|---------|-------------|
| MissingValuesSummaryPlugin | Summarizes overall missing-value statistics. |
| MissingValuesReportPlugin | Reports missing values by column. |
| PotentialMissingValuesPlugin | Detects configured missing-value indicators such as `"N/A"` or `"-"`. |
| SuspiciousMissingValuesPlugin | Detects suspicious values such as `999`, `-999`, `"UNKNOWN"`, and `"TBD"`. |
| SparseRowsReportPlugin | Reports rows with many missing values. |
| PotentialBooleanTypePlugin | Detects columns containing exactly two unique values. |

## Returns

The original `DataFrame`.

No data is modified.

## Report

Stored in

```python
df.attrs["danda_analyze_report"]
```

---

# `impute()`

Fill missing values using configurable strategies.

```python
df = df.dg.impute()
```

## Description

Missing values are filled according to the configured imputation strategy for each data type.

Supported types include:

- Numeric
- Boolean
- Category
- Text
- Datetime

Strategies include:

- Median
- Mean
- Mode
- Forward fill
- Backward fill
- Constant value

## Returns

A DataFrame with missing values filled.

## Report

Stored in

```python
df.attrs["danda_impute_report"]
```

---

# `report`

Retrieve reports generated by previous Danda operations.

```python
reports = df.dg.report
```

## Returns

A dictionary containing all available reports.

Example

```python
{
    "clean": {...},
    "optimize": {...},
    "analyze": {...},
    "impute": {...}
}
```

Reports are only included for operations that have already been executed.

For example:

```python
df = df.dg.clean()

df.dg.report
```

returns

```python
{
    "clean": {...}
}
```

# ⚙️ Configuration

`danda` is configurable, allowing you to control how cleaning and type detection behave. Every DataFrame has its own configuration, making it easy to customize behavior for different datasets.

```python
import pandas as pd
import danda

df = pd.read_csv("employees.csv")

# Disable numeric detection
df.dg.config.types.numeric_enabled = False

# Be more strict when detecting categories
df.dg.config.types.category_threshold = 0.05

# Preserve the original index when removing duplicates
df.dg.config.cleaning.remove_duplicates_ignore_index = False

df = df.dg.optimize()
```

You can inspect the current configuration at any time:

```python
print(df.dg.config.show())
```

which produces output similar to:

```text
TypeConfig
==========

Numeric Detection
-----------------

numeric_enabled                    : True
    Default     : True
    Description : Enable automatic detection and conversion of numeric columns.

numeric_threshold                  : 0.95
    Default     : 0.95
    Description : Minimum fraction of non-null values that must be successfully
                  converted to numeric before the column is considered numeric.
```

For a complete list of configuration options and examples, see the
**[Configuration Guide](docs/configuration.md)**.

# `compare_memory()`

Compare the memory usage of two DataFrames.

```python
comparison = original.dg.compare_memory(optimized)
```

## Parameters

| Parameter | Description |
|----------|-------------|
| other | The DataFrame to compare against. |

## Returns

A dictionary containing memory usage statistics.

Example

```python
{
    "before_bytes": 121746,
    "after_bytes": 74678,
    "saved_bytes": 47068,
    "saved_percent": 38.66
}
```

This method is useful for evaluating the effectiveness of optimization operations.

# Typical Workflow

A common Danda workflow consists of four stages:

```python
df = (
    pd.read_csv("customers.csv")
      .dg.clean()
      .dg.impute()
      .dg.optimize()
      .dg.analyze()
)
```

After processing, reports can be inspected:

```python
reports = df.dg.report
```

or rendered using Danda's report renderer:

```python
from danda import ReportRenderer

renderer = ReportRenderer()

print(renderer.render(df.dg.report))
```

---

# Notes

- All accessor methods return a pandas `DataFrame`.
- Analysis methods never modify the data.
- Reports are stored in `DataFrame.attrs`.
- Operations are designed to be chainable.
- Configuration is controlled through `df.dg.config`.
